# Spotify Manager

Консольный менеджер любимых треков Spotify. Приложение выбирает треки из раздела
**Liked Songs** по году добавления и переносит их в выбранный плейлист и, при
необходимости, удаляет из любимых.

## Возможности

- поиск любимых треков по году добавления;
- перенос найденных треков в плейлист по названию, URL или Spotify URI;
- создание нового плейлиста, если нужный плейлист не найден;
- пропуск дублей при добавлении треков;
- удаление любимых треков по году добавления;
- интерактивные подтверждения перед переносом и удалением;
- логирование работы приложения в файл.

## Требования

- Python 3.11 или новее;
- аккаунт Spotify;
- приложение в [Spotify Developer Dashboard](https://developer.spotify.com/dashboard).

## Зависимости

Проект использует:

- `spotipy>=2.24,<3` для работы со Spotify Web API;
- `python-dotenv>=1.0,<2` для загрузки переменных окружения из `.env`;
- `requests>=2,<3` для HTTP-запросов.

## Установка

Создайте и активируйте виртуальное окружение:

Для Unix/macOS:

```bash
python -m venv .venv
source .venv/bin/activate
```

Для Windows:

```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
# или
.venv\Scripts\activate
```

Установите проект в editable-режиме:

```bash
pip install -e .
```

После установки будет доступна CLI-команда:

```bash
spotify-manager
```

Также приложение можно запустить как Python-модуль:

```bash
python -m spotify_manager
```

## Настройка Spotify

1. Откройте [Spotify Developer Dashboard](https://developer.spotify.com/dashboard).
2. Создайте приложение или выберите существующее.
3. В настройках приложения добавьте Redirect URI, например:

```text
http://127.0.0.1:8888/callback
```

4. Создайте файл `.env` в корне проекта и вставьте значения из настроек
   Spotify-приложения:

```env
SPOTIPY_CLIENT_ID=your_client_id
SPOTIPY_CLIENT_SECRET=your_client_secret
SPOTIPY_REDIRECT_URI=http://127.0.0.1:8888/callback
```

При первом запуске приложение попросит авторизоваться в Spotify. Токен
сохраняется в файл `.spotify_token_cache`.

## Запуск

```bash
spotify-manager
```

Или:

```bash
python -m spotify_manager
```

После запуска приложение покажет текущего авторизованного пользователя и
предложит выбрать действие:

```text
1. Transfer liked tracks to playlist
2. Delete liked tracks
```

Завершить программу можно в любой момент через `Ctrl+C`.

## Перенос любимых треков

Выберите действие `1`, затем:

1. Введите год добавления треков от 2008 до текущего времени.
2. Введите название, URL или Spotify URI плейлиста, в который хотите перенести
   треки.
3. Подтвердите перенос найденных треков.
4. При желании подтвердите удаление успешно перенесенных треков из любимых.

Если по названию найдено несколько доступных для редактирования плейлистов,
приложение предложит выбрать нужный из списка.

## Удаление любимых треков

Выберите действие `2`, затем:

1. Введите год добавления треков от 2008 до текущего времени.
2. Проверьте количество найденных треков.
3. Подтвердите удаление.

Удаляются только треки из раздела **Liked Songs**, добавленные в выбранном году.

## Логи

По умолчанию логи пишутся в:

```text
logs/app.log
```

Путь к файлу и уровень логирования можно изменить через переменные окружения:

```env
LOG_FILE=logs/app.log
LOG_LEVEL=INFO
```

Если `LOG_FILE` указан относительным путем, он будет рассчитан от корня проекта.

```text
├── app.py            # Сборка приложения
├── auth.py           # OAuth-настройка Spotify
├── main.py           # Точка входа CLI
└── ui.py             # Консольный интерфейс
```
